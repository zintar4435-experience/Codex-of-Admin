# Техническое задание: набор иконок «Codex of Admin»

**Документ:** спецификация UI-иконок для веб-панели ProxyPanel (Codex of Admin)
**Версия:** 1.0
**Статус:** готов к исполнению

---

## 1. Назначение

Создать собственный набор интерфейсных иконок для панели управления и
заменить им подключаемые с внешнего CDN Bootstrap Icons.

**Зачем:**
- убрать зависимость от внешних серверов (CDN бывает медленным/заблокированным
  в ряде регионов → панель долго грузится);
- единый фирменный стиль, согласованный с логотипом-щитом «Codex of Admin»;
- всё хостится локально на сервере панели.

Набор покрывает **только продукт Codex of Admin** (панель). Иконки для
клиентского приложения Codex of Connect в это ТЗ не входят.

---

## 2. Фирменный стиль (референс)

Иконки должны быть единообразны с логотипом панели — щит с галочкой в
скруглённом квадрате (squircle), линейный, на тёмном фоне.

**Палитра бренда** (для согласованности; сами иконки красятся через
`currentColor`, см. §4):

| Роль | HEX |
|------|-----|
| Основной акцент (mint/emerald) | `#3DDC97` |
| Яркий акцент (hover) | `#5BE8AB` |
| Тёмный фон (ink) | `#0E1015` |
| Фон карточки | `#161A21` |
| Предупреждение (amber) | `#F5B544` |
| Ошибка (rust) | `#E5594E` |

Характер: спокойный, технологичный, со скруглениями (в тон щиту), без
лишней детализации.

---

## 3. Геометрия и стиль отрисовки

| Параметр | Значение |
|----------|----------|
| Формат | SVG |
| Сетка (viewBox) | `0 0 24 24` |
| Оптический отступ от краёв | ~2 px |
| Тип | контурные (line), **без заливки** |
| Толщина линии | **1.75 px** при размере 24px, одинаковая для всего набора |
| Концы линий | скруглённые (`stroke-linecap="round"`) |
| Соединения | скруглённые (`stroke-linejoin="round"`) |
| Цвет | **только `currentColor`** — никаких зашитых цветов |
| Минимальная читаемость | обязана быть чёткой при **16 px** |

**Исключение по заливке:** для семантических иконок «успех» (`cdx-ok`) и
«блокировка» (`cdx-block`) допускается заливка акцентным цветом, если так
выразительнее. Остальное — строго контур.

**Почему `currentColor`:** панель имеет тёмную и светлую темы, а пункты меню
бывают активные/неактивные. Цвет должен задаваться из CSS, а не быть зашит в
файл. Иконка обязана корректно смотреться на тёмном и на светлом фоне.

---

## 4. Формат поставки

- **Один SVG-спрайт** — файл `icons.svg`, внутри `<symbol id="cdx-…" viewBox="0 0 24 24">…</symbol>` для каждой иконки.
- **Использование в коде:**
  ```html
  <svg class="icon" width="20" height="20"><use href="/static/icons.svg#cdx-trash"></use></svg>
  ```
- Каждая иконка оптимизирована (убрать метаданные, комментарии, лишние группы; пути округлить до разумной точности).
- **Целевой вес всего спрайта:** ≤ 30 КБ.
- **Именование:** `cdx-<имя>` строго по таблице в §5.
- Дополнительно приложить таблицу соответствия `bi-* → cdx-*` (она же в §5) — для механической замены классов в шаблонах.

## 6. Критерии приёмки

- [ ] Покрыты все функции из §5 (34 иконки; синонимы объединены согласно таблице).
- [ ] Все иконки в едином стиле: сетка 24×24, линия 1.75px, скруглённые концы.
- [ ] Цвет управляется через `currentColor`; проверено на тёмной и светлой теме и в активном пункте меню.

- [ ] Имена строго `cdx-*`; приложена таблица соответствия `bi-* → cdx-*`.

## 7. Контекст использования (для дизайнера)

- Иконки выводятся в навигации и на кнопках при **14–20 px**.
- Логотип панели (щит-галочка) — отдельный ассет, уже готов; иконки должны
  быть с ним стилистически согласованы (та же скруглённость, та же «лёгкость» линии).
- Фон интерфейса преимущественно тёмный (`#0b1014` … `#161A21`), есть светлая тема.
- Технологический продукт (управление прокси-серверами) — метафоры должны быть
  понятными и «серверными», без игривости.
